# 跨项目操作习惯（cross-project operating habits）

状态：`active`
适用习惯：H01-H20

正式条目由 `project_system/02_跨项目操作习惯注册表_cross_project_operating_habits.*` 管理。本规则说明共同行为：任何任务先路由、按来源和权限决定可做范围、按工程深度选择最小充分实现、以验证和真实同步收尾。

## 通用要求

1. H01-H04：写入前必须有 `route_decision`、任务类型、责任层、必读清单/读取状态和允许/禁止范围。
2. H05-H11：按价值选择 L0-L3；大任务先 gate 后 lane；并发读取不等于并发写入。
3. H12-H14：来源先回读，供料可追溯，秘密和外部副作用先 dry-run 或阻断。
4. H15-H18：完成接力、失败路由、诚实状态和 Git 小闭环不可缺省。
5. H19-H20：唯一工作区、中文优先、英文机器字段和工具链例外必须长期遵守。

本规则是总索引；专题规则 11-18 分别拥有 route、工程线、lane、供料、文件方案、完成失败、语言和路径治理的唯一细则。
