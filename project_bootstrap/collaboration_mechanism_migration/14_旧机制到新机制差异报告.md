# 旧机制到新机制差异报告

## 总体变化

旧仓库是成熟的视频生产与 RAG 工程系统；新项目仍处于 `project_bootstrap`。本轮采取“保留控制结构、重写领域语义、删除固定实现、暂缓重工程线”的迁移策略。

| 旧机制 | 新机制 | 变化 |
|---|---|---|
| 视频项目状态动作路由 | 跨境验证状态机 | 状态改为产品事实、市场筛选、合规、B2B/B2C 验证、试点与规模决策 |
| GPT 静态包/动态事实分层 | 静态规则/动态事实/任务/证据四层 | 增加原始合同、检测、认证、平台后台与有效期 |
| 文案/素材/剪辑工作流 | 研究、事实、合规、渠道、外部动作路由 | 删除所有媒体交接件 |
| 内容/技术/发送状态分层 | 技术/证据/人工/业务/外部同步分层 | 保留不越级原则，改写业务语义 |
| Reference-to-Execution | Research Source-to-Action | 从视觉锚点改为法规、市场、竞品、客户资料到执行契约 |
| 单变量内容实验 | 渠道/报价/内容假设实验 | 不迁旧阈值，按真实样本另定 |
| 固定 RAG 和外部 AI 路线 | 技术中立的 source readback | 不绑定产品与供应商 |
| 强制提交推送 | 条件式 Git 闭环 | 仅目标成为 Git 仓库且任务授权时启用 |
| L0-L3 完整工程线 | 当前只启用 L0/L1 思想 | L2/L3 暂缓，等待重复任务和自动化价值证据 |

## 当前项目冲突

既有 `foreign_trade_low_gi_plan/` 给出了市场和 B2B/B2C 倾向，但任务 prompt 明确这些尚未成为正式结论。新底座因此不吸收这些结果，只提供人工裁决入口。

## 未完成的迁移

- 未创建正式动态事实文件，只创建模板。
- 未替换正式 `AGENTS.md`。
- 未上传 ChatGPT Project。
- 未建立 Git 仓库、自动化、schema 或 validator。
- 未用真实跨境任务验证候选机制。

## 复审判据

若候选底座不能减少重复解释、事实误升格、外部动作越权和任务漏项，应继续简化，不以文件数量作为成功。
