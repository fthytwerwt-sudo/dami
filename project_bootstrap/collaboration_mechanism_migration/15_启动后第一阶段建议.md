# 启动后第一阶段建议

以下为 `new_recommendation`，不是已批准执行计划。

## 建议目标

先完成协作底座人工审核和事实盘点，再开始任何市场或渠道任务。

## 建议顺序

1. **裁决既有业务资料权威性**：逐项检查 `foreign_trade_low_gi_plan/` 中国家、B2B/B2C、平台、健康表述和 30 天方案，标成已确认、有证据未确认、候选、否决或过期。
2. **批准最小静态规则**：只审核 `04`、`05`、`09`、`12`，删除不适合用户工作方式的字段。
3. **建立正式事实入口**：从空模板开始，只有带原始证据和确认者的内容进入正式事实。
4. **选一个低风险真实任务试运行**：建议是“产品事实缺口盘点”，只读、不外发、不定市场，用 `10` 下发、`11` 回报。
5. **复盘协作有效性**：检查是否减少误解、漏项和状态越级；只修实际暴露的问题。
6. **再决定是否 Git 化和自动化**：重复任务、多人协作或审计价值明确后，才启用 Git 闭环或 L2/L3 工程机制。

## 第一阶段验收

- 用户明确哪些旧资料是正式决策，哪些只是候选方案。
- 正式事实文件拥有证据、有效期、确认者和替代关系。
- 一次真实任务完整经过影响面、执行、验证、回报和人工复核。
- 没有市场、合规或外部动作被 AI 自动决定。

## 停止线

- 若事实权威仍冲突，停在 `human_decision_required`。
- 若需要法律/食品监管判断，转专业确认，不由 AI 给最终结论。
- 若机制维护成本大于减少的误解和风险，回退到更轻的模板。
