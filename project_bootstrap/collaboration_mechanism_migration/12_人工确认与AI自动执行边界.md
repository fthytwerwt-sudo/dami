# 人工确认与 AI 自动执行边界（候选版）

状态：`candidate_for_review`。以下不是最终政策，等待用户逐项审核。

## 必须人工确认的候选事项

- 国家和市场最终选择。
- 产品合规声明，以及医疗、健康、低 GI、控糖等功效表述。
- 定价承诺、正式报价、MOQ、交期和代理条件。
- 合同、法律文本最终签署与承诺。
- 大额付款、付费 API、广告预算和采购。
- 样品寄送、收件信息使用和物流承诺。
- 客户正式报价与关键谈判。
- 平台账号关键操作、店铺开通、上架和权限变更。
- 邮件、WhatsApp、LinkedIn 等正式外发及批量触达。
- 删除数据、覆盖正式事实、修改正式业务状态。
- 从测试进入规模化。
- 任何需要专业法律、食品监管、税务或医疗判断的最终结论。

## AI 可以自动做的候选事项

- 资料整理、字段提取、去重和数据清洗。
- 候选市场初筛与公开数据研究。
- 客户名单草拟和公开信息分类；不自动触达。
- 邮件、消息、报价说明和内容草稿；不自动发送。
- 状态报告、证据索引、风险扫描和缺口清单。
- Codex 执行单、回报模板和自动化方案设计。
- 在已授权沙箱中的低风险、可逆测试。
- 检查文件格式、链接、JSON/schema、重复项和状态一致性。

## 必须停止并请求确认

- 健康/合规表述缺原始证据或适用国家不明。
- 需要账号、密钥、付款或外部系统写权限。
- 将产生客户可见承诺或声誉影响。
- 删除、覆盖、迁移正式数据，或无法完整回滚。
- 候选方向将被提升为正式市场/渠道/商业模式。

## 授权记录最低字段

`action / target / scope / limit / valid_from / valid_until / approved_by / revocation / evidence`。
