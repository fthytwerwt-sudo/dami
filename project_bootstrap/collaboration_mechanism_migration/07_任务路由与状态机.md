# 任务路由与状态机（候选版）

## 通用路由字段

每次状态判断记录：`current_state / evidence / entry_condition_met / allowed_actions / forbidden_actions / done_when / failure_condition / next_state / human_gate`。任何状态都可因关键事实缺失进入 `blocked`，或因需要业务裁决进入 `human_decision_required`。

## 状态定义

### `project_bootstrap`

- 进入：项目尚未形成已审核的协作规则和正式事实入口。
- 允许：机制审计、模板、事实盘点、权限设计。
- 禁止：市场、平台、商业模式定案和业务外发。
- 完成：规则与事实入口获人工批准并完成一次真实任务试运行。
- 失败：静态规则与动态事实无法分离。
- 下一状态：`product_fact_collection` 或 `human_decision_required`。

### `product_fact_collection`

- 进入：协作底座已批准，产品证据清单明确。
- 允许：收集检测、配方、包装、产能、资质、报价输入。
- 禁止：无证据的健康/合规声明和客户承诺。
- 完成：关键产品事实有原始证据、有效期和责任人。
- 失败：核心证据不可得或相互冲突。
- 下一状态：`market_screening`、`compliance_validation` 或 `blocked`。

### `market_screening`

- 进入：最小产品事实足以定义筛选条件。
- 允许：公开数据研究、候选国家评分、假设比较。
- 禁止：把评分直接写成最终市场决定。
- 完成：候选短名单、证据、反证和验证计划齐全。
- 失败：数据过期、来源不可比或标准不稳定。
- 下一状态：`compliance_validation` 或 `human_decision_required`。

### `compliance_validation`

- 进入：产品与候选市场范围明确。
- 允许：法规、标签、认证、claim、平台准入核验。
- 禁止：AI 自行批准合规、健康或医疗表述。
- 完成：适用要求、证据缺口、专业/人工确认记录齐全。
- 失败：关键合规证据缺失或需专业意见。
- 下一状态：渠道验证、`blocked` 或 `human_decision_required`。

### `b2b_channel_validation` / `b2c_channel_validation`

- 进入：对应市场与最小合规边界已获人工批准。
- 允许：渠道研究、名单草拟、内容/话术草稿、沙箱测试。
- 禁止：未授权触达、报价、上架、广告或合同承诺。
- 完成：渠道假设、样本、指标、失败线和证据齐全。
- 失败：渠道不可达、成本/风险超阈值或证据不足。
- 下一状态：`lead_generation_test`、`offer_validation`、`human_decision_required`。

### `lead_generation_test`

- 进入：外发范围、对象、模板、频率和授权明确。
- 允许：仅执行已授权的小样本动作与记录。
- 禁止：扩大对象、自动群发或改变承诺。
- 完成：样本、响应、合规异常和结论可追溯。
- 失败：投诉、退信、权限或数据风险触发停止线。
- 下一状态：`offer_validation`、`blocked` 或返回渠道验证。

### `offer_validation`

- 进入：产品、报价输入和客户需求证据足够。
- 允许：获批范围内测试价值主张和交易条件草案。
- 禁止：未经人工确认的价格、交期、MOQ、功效或合同承诺。
- 完成：报价/样品/条件的人工批准与客户反馈证据齐全。
- 失败：履约或合规条件不成立。
- 下一状态：`pilot_execution` 或停止。

### `pilot_execution`

- 进入：试点范围、预算、责任、合规、回滚获批。
- 允许：严格按试点执行单实施与监测。
- 禁止：自动扩量和跨范围复制。
- 完成：试点指标、成本、风险、复盘和人工结论齐全。
- 失败：触发安全、合规、财务或客户停止线。
- 下一状态：`scale_or_stop`。

### `scale_or_stop`

- 进入：试点有完整证据。
- 允许：生成扩量、继续验证或停止建议。
- 禁止：AI 自动决定规模化。
- 完成：用户作出正式决定并记录依据。
- 失败：证据不足则退回试点或 `human_decision_required`。
- 下一状态：由正式决策指定。

### `blocked`

- 进入：关键事实、权限、能力、证据或安全条件缺失。
- 允许：诊断、补证据、提出替代方案和损失。
- 禁止：用降级结果冒充目标完成。
- 完成：阻断解除并有证据。
- 下一状态：返回原状态或 `human_decision_required`。

### `human_decision_required`

- 进入：存在业务、合规、资金、外发、删除或不可逆分支决策。
- 允许：呈现选项、证据、风险和默认安全动作。
- 禁止：AI 代替用户拍板。
- 完成：授权人明确决定并记录时间、范围和条件。
- 下一状态：按决定进入目标状态。
