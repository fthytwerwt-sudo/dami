# Codex 任务交接契约

向 Codex 下发复杂、修改、研究、合规、自动化或外部动作任务时，必须提供以下字段。不适用的字段必须明确写 `not_applicable` 并说明原因，不得默认省略。

## Goal

说明要解决的真实问题、目标状态和本轮不追求的结果。

## Context

说明当前状态、已有工件、相关历史与任务为何在此时执行。不得把历史信息默认写成当前事实。

## Source of truth

按优先级列出必须回读的正式文件、原始证据或外部权威来源，以及有效期和冲突处理方式。检索摘要不得代替原文件。

## Constraints

列出业务、合规、安全、隐私、时间、成本、仓库可见性和工具边界，并指明不能被替代路线绕过的约束。

## Implementation design

说明已确认能力、待探测能力、首选路线、替代路线与替代损失、工具或授权依赖、回滚方案和失败停止线。

## Impact check

在修改前检查真实项目根、Git 根、分支与远端、未提交改动、已有正式机制、目标路径冲突、仓库可见性、秘密与敏感数据风险。报告将受影响与不会受影响的范围。

## Allowed files

精确列出可读取、可创建、可修改、可删除和可暂存的路径。路径授权不自动授权外部动作。

## Forbidden files

精确列出不得读取、修改、删除、暂存或提交的路径。默认不得将无关本地脏改、密钥、凭据、私有来源或商业敏感内容纳入任务。

## Execution steps

用有顺序的可验证步骤说明如何从读取、判断、执行到回写。依赖任务顺序执行；只有独立任务可并行。

## Validation

列出对应每个完成声明的验证方法、命令或回读证据。必须分开技术验证、证据验证、人工复核、业务决定和外部同步。

## Done when

使用可观察、可回读的条件定义完成。若需要 Git 闭环，必须包含精确暂存、commit、普通 push、remote readback，以及本地 `HEAD`、`origin/main` 和 `git ls-remote` SHA 一致。

## Blocked if

列出必须停止的条件和对应具体阻断状态，例如事实源缺失、事实冲突、需要授权、合规证据不足、能力未验证、写入越界、需要访问密钥、公开仓库不适合敏感数据或远端回读失败。

## Output

规定交付文件、状态字段和结束回报形式。结束回报至少包含：状态、已改文件、验证证据、`Tested`、`Not-tested`、阻断项、未声称状态和下一个安全步骤。
