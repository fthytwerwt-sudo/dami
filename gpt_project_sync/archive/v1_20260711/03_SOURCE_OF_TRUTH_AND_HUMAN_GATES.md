# 事实源与人工闸门

## 三类权限分离

### 业务决策权

用户可以决定是否、何时以及按何种方式推进一项业务动作。该权限形成正式业务决定，但不会改写客观事实。

### 客观事实证明力

法规、检测、认证、合同原文、付款记录、平台后台等事实由其对应的有效原始证据证明。用户、ChatGPT 或 Codex 都不能用意愿、计划或推论改写客观证据。

### 执行权

Codex 只能在用户本轮明确授权、正式长期规则和任务允许范围交集内执行。ChatGPT 只能生成建议、裁决候选与执行单，不得自行进行外部业务动作。

## 来源有效期

每个影响决策的来源应记录：来源类型、出具日期或回读日期、适用对象与地域、版本或生效期、验证状态、是否已被正式替代。法规、平台规则、市场数据和时效性强的研究结论在使用前必须检查有效期。无法判定有效性时，标为待验证，不作为正式事实。

## 冲突裁决

1. 先判断冲突属于业务决定、客观事实还是执行授权，不使用一条简单线性排名处理所有冲突。
2. 对于客观事实，优先使用与事实类型直接对应、当前有效、可回读的原始证据。
3. 对于业务决定，使用用户当前明确决定；需要成为长期正式状态时，必须回写正式决策与当前状态文件。
4. 计划、草稿、候选方案、检索摘要、历史聊天和 AI 推论不得覆盖已验证来源。
5. 无法判定来源的适用性、有效期或替代关系时，进入 `blocked_fact_conflict` 或人工裁决，不自行合并。

## 原文件回读

检索结果、摘要、引用和二手转述只用于定位。影响事实、合规、合同、付款、正式决策或外部动作的内容，必须回读原文件或权威原始页面，并记录相关范围、日期与局限。原文件不可得时，结论只能标为待验证或阻断。

## 必须人工确认的动作

- 正式合规、健康、功效或专业声明。
- 对外报价、定价承诺、交付承诺和商业条件。
- 合同、法律文本的最终确认、签署或发送。
- 付款、采购、付费资源、广告或其他资金使用。
- 样品寄送、收件信息使用和物流承诺。
- 邮件、即时消息、社交平台消息、客户可见文本等正式触达或批量外发。
- 平台账号的开通、上架、权限变更或其他关键写操作。
- 删除、覆盖正式记录、修改正式业务状态或从测试进入规模化。

人工授权应至少记录 `action`、`target`、`scope`、`limit`、`valid_from`、`valid_until`、`approved_by`、`revocation` 和 `evidence`。授权不明确、已过期或超出范围时，对应动作必须停止。

## 公开仓库敏感数据边界

仓库为 Public 时，`business_sensitive_git_write_blocked_while_repo_public: true`。可以保存不含敏感信息的协作规则与状态标记，但不得写入供应商或客户详情、成本、报价、合同、认证原件、检测原件、个人信息、凭据、密钥或其他商业敏感数据。需要这些数据的任务应使用经授权的私有来源，并保持不进入公开 Git 历史。
