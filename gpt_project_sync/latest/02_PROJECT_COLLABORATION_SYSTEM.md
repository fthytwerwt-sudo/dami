# 项目协作系统

## 四层架构

1. **静态规则层**：保存角色、权限、任务结构、事实标签、阻断和完成真实性等长期规则。
2. **动态项目事实层**：仅记录经证据与相应权限确认的当前事实、决策、状态与安全边界。
3. **任务执行层**：每轮通过独立执行单串联需求对齐、实现设计、影响检查、权限边界、执行、验证和回报。
4. **证据与历史层**：保存原始来源、决策依据、实验结果、失败记录、历史状态和被替代方案。`latest` 只是当前入口，不覆盖历史。

## 角色分工

- **ChatGPT**：判断表面问题背后的真实卡点，组织事实与假设，收束方向，完成实现设计并生成 Codex 执行单。
- **Codex**：按正式读取链回读事实源，做影响面与人工闸门检查，在允许范围内执行、验证、回写和如实报告。
- **用户**：保留正式业务决定、合规声明、资金、合同、对外承诺、关键平台写操作、删除和规模化的最终权限。
- **外部证据源**：法规、检测、认证、合同、平台后台等客观事实由相应权威原始证据证明。

## 激活机制

正式激活 24 项机制，每轮仅触发与任务相关的机制：

- **角色、事实与冲突**：M01、M02、M03。
- **任务路由、需求与设计**：M04、M05、M06、M07、M08、M09、M10。
- **完成、状态与同步真实性**：M11、M12、M13、M14、M15、M16、M17。
- **权限、执行和复盘**：M18、M20、M21。
- **研究、实验和 Git 闭环**：M22、M23、M27、M28。

M19 当前为 `DEFERRED`：只保留工程深度要与重复任务、风险和自动化价值匹配的判断思想，不建立复杂 schema、图编排、多节点运行时或重型自动化。只有在出现可证明的重复任务与自动化收益后才重新评审。

M24、M25、M26 为 `REJECTED_NOT_MIGRATED`：视频/TTS/剪辑/卡片/审片专属流程、固定技术依赖，以及旧项目业务状态、路径和历史结论不进入本项目正式机制。

## 每轮协作闭环

`用户意图 → 任务分类 → 需求对齐 → 实现设计 → 影响面与事实源检查 → 人工闸门检查 → 授权范围内执行 → 验证与完成真实性 → 状态与证据回写 → 授权时完成 Git 闭环`

## 路由判断

- **由 ChatGPT 处理**：意图与真实卡点判断、需求收束、事实/候选/推论分层、方案比较、实现设计和执行单生成。
- **交给 Codex**：需要真实读取或修改文件、运行工具、执行验证、回写状态或完成已授权 Git 闭环的任务。
- **需要外部研究**：结论依赖会变化的规则、标准、市场、平台或专业信息，且当前证据不足。研究必须保留来源、日期、适用范围和局限。
- **必须由用户决定**：正式业务方向、合规声明、报价与对外承诺、合同、付款、样品、正式触达、关键平台写操作、删除和从测试进入规模化。

任何路由都不得将技术完成、Git 同步或草稿生成写成人工通过、业务通过或外部动作完成。
